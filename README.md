# 나만의 웹서버 만들기

## 📌 설명
이 프로젝트는 Java의 Socket을 사용하여 프레임워크 없이
HTTP 서버를 직접 구현하는 도전 과제입니다.

우아한테크코스 프리코스 오픈 미션으로 시작하게 되었습니다.

처음에는 단순히 클라이언트 연결을 받고
항상 `HTTP/1.1 200 OK`를 응답하는 작은 서버에서 시작하여,
점진적으로 HTTP 파싱, 라우팅, 정적 파일 서빙, 스레드 처리 등
웹 서버의 핵심 기능들을 직접 구현하는 것을 목표로 합니다.

## 💡 계기
프레임워크가 자동으로 해주던 HTTP 처리 과정을 직접 구현하며
웹의 본질과 네트워크 동작 원리를 이해하는 것이 목표입니다.

## 🎯 목표
- [ ] TCP 소켓 서버 구현
- [ ] HTTP 요청 라인/헤더 파싱
- [ ] 라우팅 및 기본 핸들러
- [ ] 정적 파일 서빙
- [ ] 에러 처리 (404/500 등)
- [ ] 멀티스레드 처리

## 🔄 흐름
Client -> Socket -> Request 파싱 -> Router -> Handler -> Response


## 🗂️ 파일 구조
my-webserver

## 고찰
원래 이번 프리코스는 전에 진행했던 SpringBoot + Java로
웹사이트를 만든 프로젝트를 Kotlin으로 리팩토링 하는 것을 해보려고 했다.

하지만 개발을 해오면서 궁금했던 점이 있었다.
SpringBoot 내부에서 동작하는 웹서버(톰캣)는 도대체 어떤 방식으로 요청을 받아서 처리할까?
이 의문을 이번에 해결하고 싶어서 이 주제를 선정하게 되었다.

물론 이 프로젝트가 나에게 너무 어려운 도전일지도 모른다고 생각한다.
네트워크와 HTTP를 처음부터 직접 구현한다는 점에서 막막했기 때문이다.

하지만 `도전`이라는 우아한테크코스의 키워드를 생각해보면
지금 나에게 필요한 건 완성된 프로젝트가 아닌 낯선 영역에도 부딪쳐보는 경험이라고 생각한다.
아무것도 모를 때 백엔드 개발, 클라우드 서비스를 활용한 서버와 DB 구축을 처음 했을 때가 생각난다.

실패하더라도 그 과정에서 하나라도 얻어가보자.
